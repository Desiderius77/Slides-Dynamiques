Introduction

Le sous-titrage est une technique assez ancienne pour permettre d'ajouter de l'information à une séquence vidéo

  • Il est particulièrement important
    • Pour internationaliser un contenu : sous-titre dans une langue différente de celle qui est parlée
    • Pour aider à la compréhension des personnes malentendantes ou sourdes
      • Fourniture du contenu du dialogue parlé
      • Fourniture d'indications sur qui parle ou sur l'environnement sonore de la scène ( audiodescription)
    • Pour le divertissement : karaoké
    • Pour donner des accès vers des ressources supplémentaires

Techniques

    • Augmentation des besoins en accessibilité => plus de sous-titres pour plus de supports multimédia

Vers la définition d'un nouveau format
  • L'arrivée de HTML5 a précipité les choses et permis à un Groupe de Travail de réfléchir à la création d'un nouveau langage pour le sous-titrage
  • Ce langage doit permettre
    • d'ajouter des sous-titres à un flux vidéo
    • de chapitrer les sous titres
    • de formatter les sous-titres
    • de sémantiser les sous-titres
    • de permettre d'y accéder par Javascript
  • Ce groupe de travail s'est rassemblé sous l'égide du W3 Consortium pour mettre au point la spécification du langage WebVTT ( Web Video Text Tracks )
    • L'objectif est de standardiser ce langage en tant que W3C Recommandation
    • 47 personnes représentant Apple, l'INA, VLC , Mozilla, Adobe, Opera, beaucoup d'Universités dont .... Evry

Structure générale du fichier WebVTT
  • Le langage WebVTT s'est inspiré du format SRT
  • On retrouve donc la forme générale du fichier SRT
    • Une ligne donnant un titre de chapitre par exemple ( indexation)
    • Une ligne donnant les Time Code de début et de fin de la séquence de sous-titrage
    • Une ligne donnant le texte à afficher
  • Exemple 
                WEBVTT
              
              Introduction
              00:00.000 --> 00:01.000
              Sous-titre de la première minute
              
              Sujet principal
              00:00.000 --> 00:01.000
            Sous-titre de la deuxième minute
Module Multimédia 7/28 M2 MIAGE APP

Structure générale du fichier WebVTT ( suite )
  • Le fichier commence obligatoirement par le chaine de caractères WEBVTT en début de ligne
  • Les différents sous-titres sont séparés par une ligne blanche
  • Un paquet "Titre+ time-codes + sous-titres"
    • est donc matérialisé par une séparation d'une ligne blanche au-dessus et d'une ligne blanche au dessous
    • prend le nom générique de cue
  • Les Time Codes sont exprimés obligatoirement sous la forme [Heures:]Minutes:Secondes.Millisecondes avec les valeurs 00 ou 000 obligatoires
  • Le texte du sous-titre peut s'étaler sur plusieurs lignes tant qu'il n'y a pas de ligne vide

Contenu des cues

Le contenu présent dans les cues peut être

  • du texte ordinaire
  • du texte HTML ( ie contenant des balises HTML comme <p> <b> ou <i>
  • un fichier JSON
  • Exemple 1 
                L'université d'Evry
              00:00.000 --> 00:01.00
              <p> Cette université fait partie des <b>nouvelles</b> universités d'Ile de France </p>
  • Exemple 2 
                miage
              00:03:45.000 --> 00:04:14.000
              {
              "title": La MIAGE d'Evry
              "description": "Elle est spécailisée dans les technologies du Web"
              "src": "miage_evry.jpg"
              "href: "http://www.univ-evry.fr"
              }

Mise en page des cues

Le texte des cues peut être mis en page par 5 opérateurs

  • line : positionnement en lignes
  • align : alignement
  • position : position des textes
  • size : taille des textes
  • vertical : direction d'écriture verticale
Module Multimédia 10/28 M2 MIAGE APP

Mise en page des cues (suite)
  • line
    • peut être un entier ( numéro d'ordre de la ligne )
    • peut être un pourcentage ( par rapport à la hauteur totale de la vidéo)
  • align
    • start, middle, end, left, right
  • position
    • Position du texte en pourcentage par rapport à la largeur de la vidéo
  • size
    • Taille du texte en pourcentage
  • vertical
    • Définit la direction d'écriture du texte vertical : lr ( vertical ascendant ) ou rl ( vertical descendant )

Utilisation avec HTML5
  • Pour pouvoir utiliser ces sous-titres, il faut les associer à une source vidéo
  • C'est le rôle de la balise <track> de HTML5
  • track a les attributs suivants
    • kind indique un type parmii la liste
      • subtitles : pour des sous-titres
      • captions : pour des sous-titres enrichis d'indications sur l'environnement sonore
      • descriptions : pour des textes destinés à être lus par une machine
      • chapters : pour ajouter une structure de navigation
      • metadata : pour ajouter des métadonnées ( qui ne seront pas affichées )
    • scrlang indique la langue des sous-titres
    • src localise le fichier
    • label donne une étiquette pour une possible indexaton

Utiliser Track et WebVTT en Javascript

Les élements audio et vidéo de HTML5 ont une propriété textTracks qui renvoie une liste TextTrackList dont tous les membres sont des TextTrack correspondant à un élément <track> 

          var videoElement = document.querySelector("video");
            var textTracks = videoElement.textTracks; // un pour chaque élément track
            var textTrack = textTracks[0]; // le premier élément track
            var kind = textTrack.kind // par exemple "subtitles"
            var mode = textTrack.mode // 0 (TextTrack.OFF dans la spec, TextTrack.DISABLED dans Chrome),
                                         1 (TextTrack.HIDDEN) ou 2 (TextTrack.SHOWING)

Chaque TextTrack, à son tour, a une propriété cue qui renvoie une TextTrackCueList dont les membres sont des cues.
Les données des cues peuvent être accédées par des propriétés telles que startTime, endTime et Text 

          var cues = textTrack.cues;
            var cue = cues[0]; // le premier cue du fichier WebVTT
            var cueId = cue.id // l'identificateur donné ua cue dans le fichier WebVTT
            var cueText = cue.text; // Le texte du cue ( ou le JSON )

Ajouter un cue

Où en est le support dans les navigateurs Web ?
  • IE 10
    • Support dansPlatform Preview 4 le 30 Novembre 2011

  • Google Chrome
    • Support de WebVTT depuis la version 18 en Novembre 2011

    • Apple a publié le support de WebVTT avec la version 6 en Juillet 2012
  • Opera
    • Support dans Opera Next en Août 2012
  • Firefox
    • Pas encore de support dans une version officielle
    • Premier développement partiel
